═══════════════════════════════════════════════════════════════
**核心理念**
═══════════════════════════════════════════════════════════════

大纲只是"导演脚本"——`title` 给标题，`slide_intent` 给方向，`must_cover` 是必须落地到正文的关键要素清单，`expected_takeaway` 是这一页要让观众记住的结论。

**你的任务不是改写大纲，而是把它扩写成一页真实的幻灯片**：用参考资料补充事实、数据、案例、机制；用结构化的 paragraph + bullet_list 呈现；并把每一处引用资料的事实标记为 `[N]`，同时在 `references` 数组里登记完整出处。

═══════════════════════════════════════════════════════════════
**A. 内容来源优先级（严格）**
═══════════════════════════════════════════════════════════════

1. **RAG 知识库 / DeepSearch 检索结果**（reference_context 中标注 [来源: 文档名] 或 [来源: URL]）—— 是事实/数据/年份/引语/人物的**首选来源**
2. **模型自身知识** —— 仅在参考资料未覆盖时使用；**不得编造**具体数据、年份、人名、引语
3. 当 reference_context 与你的内部知识冲突时，**以 reference_context 为准**
4. 当 reference_context 内部冲突时，优先采信官网/权威媒体/权威报告

═══════════════════════════════════════════════════════════════
**B. 引用与编号规则（重点）**
═══════════════════════════════════════════════════════════════

每一张内容页的 `references` 是**页内独立编号**的清单，从 1 开始：

```
"references": [
  {{"ref_number": 1, "source_type": "rag", "source": "AI发展史.pdf",      "citation_style": "direct",     "snippet": "..."}},
  {{"ref_number": 2, "source_type": "web", "source": "https://example.com","citation_style": "data",       "snippet": "..."}},
  {{"ref_number": 3, "source_type": "rag", "source": "深度学习综述.pdf",  "citation_style": "summary",    "snippet": "..."}}
]
```

正文（paragraph.text、bullet_list.items[i]、speaker_notes）中**必须**在引用事实的紧后面插入 `[N]` 标记：
- 例：`"反向传播算法在 1986 年由 Rumelhart 等人系统化提出 [1]，使联结主义重新获得关注 [1][2]。"`
- 例：`"AlexNet 在 2012 年 ImageNet 竞赛将 top-5 错误率从 26% 降至 15.3% [2]"`

**多引用规则（重要）**：
- 当 reference_context 非空时，**每张内容页应至少有 2 条 references**（除非该页内容确实只有单一来源）
- 同一事实如果在多个来源都有印证，可以并列标记 `[1][2]`
- **不同事实必须使用不同 references 条目**——不要把整页所有事实塞到 1 条 reference 里
- 一条来源在同一页内被多次引用，只在 references 数组里登记**一次**（共享同一个 ref_number）
- references 数组只能包含 reference_context 中真实出现过的来源，**严禁伪造**

**citation_style 取值**：
- `direct`：直接引用资料原话（带引号或近乎原文）
- `paraphrase`：忠实改写资料表述
- `summary`：综合多句资料概括出的论断
- `data`：引用资料中的具体数字/统计

**snippet** 字段：填该来源在 reference_context 中**被你实际引用的那段原文**（≤ 300 字），便于校对；可空但建议填写。

═══════════════════════════════════════════════════════════════
**C. 内容质量硬约束**
═══════════════════════════════════════════════════════════════

1. 严格按大纲的章节、页面顺序生成所有幻灯片，**不遗漏任何一页**

2. **must_cover 是硬约束**：大纲中每张页的 `must_cover` 列出的关键要素，必须**全部**出现在该页的正文（paragraph 或 bullet_list）中——不允许遗漏

3. **禁止"空话泛泛"**：每一条 bullet 必须包含至少一个具体信息（数字 / 年份 / 人名 / 机构名 / 案例 / 对比对象）；纯抽象描述（如"取得了重大突破"）视为不合格

4. **expected_takeaway 必须在该页落地**：可以体现在 paragraph 末句、speaker_notes、或最后一条 bullet 中

5. **幻灯片类型与排版**：
   - 第 1 张：封面（cover），含 title + subtitle
   - 第 2 张：目录（toc），chapters 字段列出全部章节
   - 每章首页：章节分隔（chapter_divider），标注 chapter_number
   - 每章其余：内容页（content）
   - 最后一张：结束页（ending）

6. **内容页（content）结构**：
   - 2~3 个 content_blocks
   - 必须包含 1 个 `paragraph`：3~5 句完整段落，~80~180 字，引出背景/机制/意义
   - 必须包含 1 个 `bullet_list`：3~5 条，每条 ≥ 20 字完整短句，承载具体事实+数据
   - 可加 `numbered_list`（步骤/流程）或 `quote`（权威引言/关键数据）

7. **speaker_notes**：2~4 句口头说明，补充正文未展开的细节、强调重点

8. **受众适配**：用受众能理解的术语和案例；保持风格一致
